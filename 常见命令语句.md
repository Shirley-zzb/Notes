# 常见命令语句

1. 导出/安装requirement.txt文件
```python
pipreqs . --encoding=utf8 --force
pipreqs --ignore .venv --force 虚拟环境需要忽略
pip install -r requirements.txt
```
2. 导出当前环境的配置
```python
pip freeze > requirements.txt
```

3. 配置镜像源

   ```
   conda config --show channels 显示镜像源
   添加清华镜像源
   conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/
   conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/main/
   conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
   设置搜索时显示通道地址
   conda config --set show_channel_urls yes
   
   一般在使用conda install安装时会出现包无法找到或者安装失败的情况，此时可以使用pip install来尝试安装(以opencv为例)：pip install opencv
   若安装速度过慢可单独指定安装镜像加快安装:pip install opencv -i https://mirrors.aliyun.com/pypi/simple/
   ```

4. 更改默认安装包路径

   ```
   查看环境和位置
   conda info --envs
   查看conda的配置, 看envs_dirs,有3个路径，第一个默认
   conda config --show
   更改默认安装路径
   # 增加环境路径
   conda config --add envs_dirs [本地安装路径]  # envs_dirs是变量，照抄
   # 检查是否增加成功
   conda config --show 
   # 删除环境路径
   conda config --remove envs_dirs [安装路径] # envs_dirs是变量，照抄
   如果创建的环境还是在.conda文件中，则说明env文件夹的权限不够，打开即可
   也要注意pkgs的权限
   ```

5. 创建环境

   ```
   conda create -n your_env_name python=X.X 
   # 或
   conda create --name your_env_name python=X.X
   
   conda create --name=labelme python=3.8
   conda create -p D:\miniconda3\envs\labelme python=3.8
   
   conda install [package] #安装包
   conda remove [package]  #删除包
   conda update [package]  #更新包
   conda remove -n your_env_name --all # 删除环境
   conda create --name newenv --clone myenv # 克隆环境
   
   启动/退出新环境：
   source activate labelme
   # 或者
   conda activate labelme
   conda deactivate
   
   列出所有环境：
   conda env list
   conda info -e
   ```

6. 环境迁移

   ```
   要导出当前的环境，你可以使用 conda env export 命令。通常，环境信息会导出到一个 YAML 文件中。假设你要导出一个名为 myenv 的环境，命令如下：
   conda env export -n myenv > myenv.yml # 环境所有依赖和版本信息
   conda env export -n myenv > environment.yml
   导入环境
   conda env create -f envirment.yml
   
   导出包
   conda list --export > pkgs.txt
   conda list -e > requirements.txt
   导入安装
   conda install --yes --file requirements.txt
   
   导出位置在用户名里面！
   
   在Conda环境中，如果你想要导出一个仅包含所有依赖包的环境文件，而不包括当前环境中未被引用的包，你可以使用以下命令：
   conda env export --from-history > environment.yml
   这个命令会创建一个名为environment.yml的文件，其中只包含当前环境中直接或间接依赖的包。
   
   如果你想要导出到一个指定的文件名或路径，可以这样做：
   conda env export --from-history > my_environment.yml
   在这个例子中，my_environment.yml将会替换environment.yml作为输出文件。
   
   请注意，这种方法只会导出依赖关系，不会包含环境中的特定版本信息。如果你需要精确的版本信息，可以使用以下命令：
   conda env export > my_environment.yml
   这将会导出当前环境中所有包的精确版本信息。
   提示：AI自动生成，仅供参考
   ```


 7. Git常用命令操作

    ```python
    设置用户名
    git config [--global] user.name [name] 没有就是查看，有就是设置
    git config [--global] user.email [email]
    git config [--global] user.passwork [pwd]
    
    查看账户信息
    git config --list
    
    查看所有分支（-a）
    git branch (-a)
    创建本地分支
    git branch [branchname]
    创建远程分支 git push 远程主机名 本地分支名：远程分支名
    git push origin dev:dev
    切换分支
    git checkout branchname
    合并分支
    git merge branchname
    删除分支
    git branch -d branchname
    拉取远端的并在本地创建一个分支与之对应
    git checkout -b dev origin/dev
    
    
    在Git中，可以使用以下命令来查看远程仓库：
    git remote -v：该命令会列出所有已配置的远程仓库的名称和URL。这是一种常用的方式，它会显示远程仓库的名称和相应的URL链接。
    git remote show <remote-name>：这个命令用来显示指定远程仓库的详细信息，包括它的URL、分支信息、最新commit等。
    git ls-remote：这个命令会显示每个远程分支的引用，以及与之相关的SHA-1哈希值。它不会实际更新你的仓库，而只会显示远程仓库的信息。
    注意：以上方法都是在命令行中使用Git进行操作。
    
    
    另一台电脑上建立一个远程仓库与已建立的仓库同步
    git init 先初始化仓库再添加
    git remote add origin url
    git fetch
    忽略SSL证书错误
    git config --global http.sslVerify false
    登录的时候如果一直显示超时或者打不开，不要选网页登陆，可以选那个code登录
    
    要清除当前的Git仓库的远程仓库配置，你可以使用以下命令：
    git remote remove origin
    这里的origin是远程仓库的默认名称。如果你的远程仓库有不同的名称，请用相应的名称替换origin。

    4、填写标题，将公钥粘贴

    地址就可以了，没有一点问题！
    服了！
    
    ```
